# MemoAuto 产品分析与迭代建议

本文基于 `GEMINI.md`、`src/App.jsx`、`src-tauri/src/db.rs` 和 `src-tauri/src/lib.rs` 对当前产品进行复盘，重点判断各模块是否满足 C 端用户使用，以及功能链路是否闭环。

## 产品定位

MemoAuto（备忘录Auto）是一款本地优先的个人生产力工具，面向个人开发者和独立工作者，帮助用户快速捕捉信息、整理工作任务、管理项目并沉淀知识。

当前技术架构：

- 前端：React 18、Vite、Ant Design 5
- 桌面端：Tauri v2
- 后端：Rust
- 数据库：SQLite / rusqlite
- 导出：rust_xlsxwriter

建议继续收敛为：

> 面向个人开发者/独立工作者的本地优先工作备忘录。

这个定位下，产品主路径应是：

1. 快速收集
2. 整理为便笺、任务或知识
3. 关联项目
4. 执行和更新状态
5. 搜索复用或导出复盘

## 当前版本关键进展

从 `GEMINI.md` 和代码看，当前版本已经完成了部分 P0/P1 能力：

- 已支持全局搜索，可按标题、内容和项目名检索。
- 已支持编辑条目，可修改标题、内容、分类、类型、状态、优先级和项目。
- 已引入 `projects` 数据模型，不再只依赖任务里的 `project_name` 聚合。
- 项目中心已支持新建项目，并展示项目下待整理任务提醒。
- 收集箱、项目中心、知识库均有空状态提示。
- 工作流动态已改名为“最近动态”，弱化了原先“工作流审计”的误导。

这些改动让产品从“只能记录”的原型，推进到了“记录 -> 整理 -> 项目化管理”的初步闭环。

## 模块闭环评估

| 模块 | 当前满足度 | 已具备能力 | 主要缺口 |
| --- | --- | --- | --- |
| 快速收集箱 | 中高 | 新增、搜索、编辑、分类、类型选择、关联项目、转待办 | 缺少归档、快捷键、提醒/到期时间、批量整理 |
| 工作项目中心 | 中高 | 项目实体、新建项目、项目任务列表、进度、待整理提醒 | 缺少项目编辑、项目归档、项目详情、删除项目后的任务处理策略 |
| 最近动态 | 中低 | 按任务更新时间展示最近状态 | 没有真实 activity log，不能复盘具体操作 |
| 个人知识库 | 中 | 知识条目展示、搜索、编辑、复制 | 缺少标签、分类、代码高亮、收藏/置顶 |
| 账号管理 | 低且高风险 | 新增、展示、复制用户名和密码 | 密码明文存储，缺少主密码、加密、自动锁定 |
| Excel 导出 | 中 | 全量任务导出 | 缺少按项目、时间、状态筛选；字段仍偏内部枚举 |

## 主要产品问题

### 1. 主路径已初步形成，但还没有“归档”终点

当前产品已经能完成：

`记录 -> 编辑整理 -> 关联项目 -> 转为待办/进行中/完成`

但完成之后缺少归档层。任务状态目前主要是 `inbox`、`todo`、`doing`、`done`、`fixed`，项目状态有 `active`、`archived`，但前端还没有成体系的任务归档和项目归档操作。

影响：

- 长期使用后，完成项会持续堆在列表里。
- 用户无法区分“已完成但仍可见”和“已归档不再打扰”。
- 项目生命周期没有真正闭合。

建议：

- 增加任务归档状态或归档字段。
- 项目中心支持归档项目，而不是只删除项目。
- 首页增加“已完成/已归档”入口或筛选。

### 2. 项目管理已实体化，但项目生命周期还不完整

代码中已经新增 `projects` 表，并支持 `get_projects`、`add_project`、`delete_project`。这是明显进步，但项目仍缺少完整生命周期管理。

当前缺口：

- 不能编辑项目名称和描述。
- 不能归档项目。
- 删除项目没有看到与任务关系的处理策略。
- 项目没有独立详情页，信息仍集中在一张卡片里。
- 项目进度只按任务数量计算，缺少优先级、截止时间、里程碑等维度。

建议：

- 增加 `update_project`。
- 删除项目时提示“保留任务并移除关联”或“同时删除任务”。
- 项目详情页展示任务、待整理项、最近动态和导出入口。
- 项目支持 active / archived 的完整切换。

### 3. 搜索已补齐，但筛选体系还不够

当前 `searchText` 支持按标题、内容、项目名搜索，并在多个模块复用。对轻量工具来说这是必要能力，但还不足以支撑长期数据增长。

当前缺口：

- 没有按类型筛选：便笺、任务、知识。
- 没有按状态筛选：待整理、待办、进行中、已完成。
- 没有按项目筛选的统一入口。
- 搜索状态在不同页面之间复用，可能出现切换模块后仍保留上一个搜索词的困惑。

建议：

- 在列表视图增加类型、状态、项目筛选。
- 切换模块时明确是否保留搜索条件。
- 搜索结果页可展示命中来源，避免用户不知道结果来自哪个模块。

### 4. 最近动态仍不是真正的操作历史

当前“最近动态”按 `updated_at` 展示任务状态，这比“工作流动态”的命名更准确，但它仍不是可复盘的动态系统。

用户真正需要的动态包括：

- 创建了什么任务
- 从什么状态改到什么状态
- 修改了哪些字段
- 哪个项目新增了任务
- 什么时候导出过数据
- 删除了什么内容

建议：

- 新增 `activity_log` 表。
- 每次新增、编辑、状态变更、删除、导出时写入日志。
- 最近动态按动作类型、项目和日期筛选。

### 5. 知识库已有基础找回能力，但沉淀能力仍弱

知识库现在支持搜索、编辑和复制，已经能满足轻量记录。但作为长期知识沉淀模块，还缺少结构化能力。

当前缺口：

- 没有标签。
- 没有分类。
- 没有收藏/置顶。
- 代码块只是 `pre` 展示，没有语言标识和高亮。
- 没有来源链接、适用项目、更新时间说明。

建议：

- 增加 tags 字段。
- 支持按标签和项目筛选知识。
- 支持置顶常用知识。
- 优化代码块展示和复制。

### 6. 账号管理仍是最高风险模块

账号管理当前能新增账号、复制用户名、复制密码，但数据库中 `accounts.password` 是明文字段。这和界面文案“安全存储您的平台凭据”不匹配。

这是当前产品最大的 C 端信任风险。

上线前建议二选一：

- 暂时隐藏账号管理模块。
- 或补齐主密码、加密存储、自动锁定、复制确认和安全说明。

最低要求：

- 密码不可明文存储。
- 默认不展示密码。
- 复制密码前有明确确认或解锁动作。
- 应用切后台或超时后自动锁定。

### 7. 导出能力可用，但还不是场景化报表

当前 Excel 导出是全量任务导出，适合备份，但不一定满足用户真实复盘场景。

当前缺口：

- 不能只导出某个项目。
- 不能按时间范围导出。
- 不能只导出未完成或已完成任务。
- 导出内容中的分类、类型、状态仍是内部枚举，不够中文化。

建议：

- 项目详情内支持“导出当前项目”。
- 增加导出前筛选和预览。
- 导出字段使用中文值，例如“个人/工作”“待办/进行中/已完成”。

## 推荐优先级

### P0：处理账号安全风险

账号管理要么隐藏，要么加密。这个模块涉及用户凭据，风险高于其他体验问题。

建议交付：

- 主密码
- 本地加密存储
- 自动锁定
- 复制密码确认
- 安全边界说明

### P1：补齐归档闭环

让任务和项目从“完成”走向“归档”，避免长期使用后列表失控。

建议交付：

- 任务归档
- 项目归档
- 已归档视图
- 完成项批量归档

### P1：完善项目生命周期

项目已经有实体模型，下一步应补齐管理能力。

建议交付：

- 编辑项目
- 归档项目
- 项目详情页
- 删除项目时处理关联任务

### P2：升级动态为 activity log

让最近动态从“更新时间列表”变成真正可复盘的操作记录。

建议交付：

- activity_log 表
- 状态变更记录
- 编辑记录
- 删除记录
- 导出记录

### P2：增强知识库结构化能力

让知识库从“能搜到”变成“能长期管理”。

建议交付：

- 标签
- 分类
- 置顶
- 代码块高亮

### P2：导出报表场景化

让导出从备份工具升级为复盘工具。

建议交付：

- 按项目导出
- 按状态导出
- 按时间范围导出
- 中文化导出字段

## 一句话结论

MemoAuto 当前已经不再是单纯的记录原型，搜索、编辑、项目实体和待整理提醒让核心路径初步闭合。下一阶段最关键的是处理账号安全风险，并补齐“完成 -> 归档 -> 复盘”的终点能力，让它真正成为可长期使用的 C 端本地生产力工具。
